# SignalArc Documentation

SignalArc is an Arc-native prediction market infrastructure platform for USDC-settled event markets, market intelligence, resolver workflows, and agent-readable APIs. It is an API-first Web3 application stack for creators, institutions, developers, and AI agents.

SignalArc is not positioned as a clone of any existing prediction market product. The repository currently contains a live web app, a modular local development stack, Arc Testnet onchain deployment, and deployment planning artifacts.

## Current Status

| Area | Status |
| --- | --- |
| Frontend | Web app is live at https://signalarc.fun, and the Next.js app can also run locally with `pnpm dev:web`. |
| Backend API | Go/Chi API runs locally with PostgreSQL through Docker. |
| Database | PostgreSQL schema migrations are complete through version 14 in local development state. |
| Smart contract | `SignalArcMarket` is deployed on Arc Testnet only. |
| Circle integration | Current Circle integration is limited to USDC/testnet USDC flow behavior. |
| Settlement scope | Testnet/prototype only; no mainnet production settlement is claimed. |

## Implemented Capabilities

- Local backend and PostgreSQL can run through Docker Compose.
- Frontend can run manually with the monorepo `pnpm dev:web` script.
- Backend exposes health, readiness, schema validation, market, trade-intent, position, resolution, settlement, agent-market, and Arc contract metadata endpoints.
- Arc Testnet contract reference is available through the backend and frontend constants.
- Current frontend includes external wallet connection and Arc Testnet transaction flow for USDC/testnet USDC approval plus `openPosition`.
- Current frontend wallet execution is Arc Testnet prototype behavior only and does not use real funds.

## Current Limitations

- No Arc mainnet deployment is present.
- No production custody or production settlement approval is claimed.
- No audit is claimed.
- Circle Wallets, Paymaster, CCTP, and Gateway are not integrated in the current repository.
- No API authentication, API key enforcement, paid access, autonomous trading, or production SLA is implemented.
